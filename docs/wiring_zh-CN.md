本文说明已测试原型机使用的接线方式。

## 重要安全提示

- 修改接线前请断开电源。
- ESP32-S3、OLED、编码器、SD 卡和打印机接口必须使用公共地。
- 热敏打印机应使用满足峰值电流要求的独立电源。
- 不要使用 ESP32-S3 的 3.3 V 引脚给热敏打印机供电。
- 不要把 5 V 的打印机 TX 信号直接接入 ESP32-S3 GPIO。
- 接线前应根据打印机丝印或说明书确认接口顺序，不要仅按插座左右位置猜测。

## OLED

OLED 使用 I2C：

```text
SDA = GPIO4
SCL = GPIO5
地址 = 0x3C
时钟 = 400 kHz
```

部分 SSD1306 模块使用 `0x3D` 地址。如果屏幕无法识别，请检查并修改固件地址。

外接 OLED 使用 GPIO4 和 GPIO5，避免占用开发板板载 I2C 总线。

## EC11 旋转编码器

已测试配置：

```text
A  = GPIO6
B  = GPIO7
SW = GPIO15
C  = GND
```

输入使用上拉，按键低电平有效。

如果旋转方向相反：

- 交换 A、B 接线；或
- 在软件中反转正交解码方向。

当前软件约将 4 个原始跳变合成为一个定位格。

## SD 卡

已测试开发板的 TF 卡使用 SPI2：

```text
Host = SPI2_HOST
SCLK = GPIO12
MOSI = GPIO11
MISO = GPIO13
CS   = GPIO2
```

不要重复初始化同一条 SPI 总线。当前 SD 挂载代码会初始化总线并挂接 SD-SPI 设备。

## 热敏打印机

最小单向打印接线：

```text
ESP32-S3 GPIO17 → 打印机 RXD
ESP32-S3 GND    → 打印机 GND
```

仅打印时不需要连接打印机 TXD。

已测试串口参数：

```text
UART1
9600 波特率
8 数据位
无校验
1 停止位
无硬件流控
```

### 检查打印机逻辑电平

使用万用表测量：

```text
打印机 TXD 相对于打印机 GND 的电压
```

UART TX 在空闲时通常为高电平。

- 约 3.3 V：通常可接入 ESP32-S3 RX。
- 约 5 V：接入 ESP32-S3 RX 前必须使用电平转换或分压。
- 只需要单向打印：打印机 TXD 可以不接。

打印机 RXD 是输入端，悬空时的电压不能可靠表示其逻辑电平标准。

## 电源建议

热敏打印机在加热打印时的电流远高于待机状态。

- 使用打印机厂家规定的电压。
- 电源应具有足够的峰值输出能力。
- 电源线应尽量短，并具有足够线径。
- 打印机电源应与 ESP32-S3 的 3.3 V 电源轨分离。
- 在低阻抗位置将两者 GND 相连。

供电不足可能出现：

- 打印过程中打印机复位
- 字迹过浅
- 只打印部分横线
- 走纸反复启停
- 地弹或电源扰动导致 ESP32-S3 复位

## 引脚冲突检查

当前分配不存在模块间重叠：

```text
OLED：    GPIO4、GPIO5
EC11：    GPIO6、GPIO7、GPIO15
SD 卡：   GPIO11、GPIO12、GPIO13、GPIO2
打印机：  GPIO17、GPIO18
```

增加新外设时还应注意：

- GPIO19 和 GPIO20 可能用于原生 USB。
- GPIO0 通常用于启动模式选择。
- 部分引脚可能具有开发板专用功能或启动绑定位功能。
